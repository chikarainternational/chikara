学習ポイントの構造 Learning point structure

各日別ページの学習ポイントは <details>/<summary> によるネイティブアコーディオンです。JavaScript不要。

Learning points use native <details>/<summary> accordions. No JavaScript required.

<details class="learn-item">
  <summary>
    <span class="learn-num">1</span>
    <div class="learn-summary-text">
      <span class="learn-phrase">I told you.</span>  <!-- 常時表示 -->
      <div class="learn-jp">昨日言ったでしょ</div>    <!-- 常時表示 -->
    </div>
    <span class="learn-arrow">▼</span>
  </summary>
  <div class="learn-body">
    <div class="learn-desc">tell + 人 で「人に言う」</div>  <!-- 開いたとき -->
    <div class="learn-examples">
      <div class="learn-ex">I told you last night.</div>
      <div class="learn-ex-jp">昨夜言ったじゃない</div>
    </div>
  </div>
</details>
クラスClass 表示タイミングVisibility 内容Content
.learn-phrase常時Always英語フレーズEnglish phrase
.learn-jp（summary内）常時Always日本語訳Japanese translation
.learn-desc展開後When open文法説明・補足Grammar / notes
.learn-ex展開後When open例文（英語）Example sentence (English)
.learn-ex-jp展開後When open例文の日本語訳Example sentence (Japanese)

リスニングポイントの構造 Listening point structure

リスニングポイントは常時表示のシンプルなリストです。

Listening points are a simple always-visible list.

<div class="listen-card">
  <div class="listen-item">
    <div class="listen-en">Did you</div>          <!-- 英語フレーズ -->
    <div class="listen-kana">ディジュ</div>        <!-- カタカナ発音 -->
    <div class="listen-note">d が弱くなる</div>    <!-- 変化の説明（任意）-->
    <div class="listen-example">Did you get it?</div> <!-- 例文（任意）-->
  </div>
  ...
</div>
.listen-note と .listen-example は省略可です。フレーズとカタカナだけでも成立します。
.listen-note and .listen-example are optional. Phrase + kana alone is sufficient.

list.html（全一覧ページ）の編集 Editing list.html (full review page)

list.html は各日別ページと独立したコンテンツを持ちます。日別ページを変更した場合は list.html も同じ内容に更新してください。

list.html holds independent copies of all content. If you update a day page, update list.html with the same changes.

同期忘れに注意。日別ページと list.html は自動同期されません。片方だけ変更すると内容がズレます。
Manual sync required. Day pages and list.html are not automatically synced. Editing one without the other causes content inconsistency.

YouTubeバナーの変更 YouTube Banner

全19ページのYouTubeバナーは page/js/yt-banner.js 1ファイルだけで管理しています。チャンネルを変更する場合はここだけ編集してください。

The YouTube banner across all 19 pages is managed in a single file: page/js/yt-banner.js. Edit only this file to update the channel.

var YT = {
  channelUrl:   "https://www.youtube.com/@SoCal",       // チャンネルURL
  channelHandle:"@SoCal",                                // ハンドル（@〜）
  channelName:  "SoCal",                                 // チャンネル名
  avatarUrl:    "https://yt3.googleusercontent.com/...", // アイコン画像URL
  bannerUrl:    "https://yt3.googleusercontent.com/...", // バナー画像URL
  label:        "英語コーチChikara",                     // バナー内タイトル
  note:         "10,000人以上の日本人を..."              // バナー下説明文
};

バナーの構成は上下2段です。

The banner has two sections:

  • 上段：チャンネルバナー画像（背景）
  • 下段：アバター画像 + タイトル + ハンドル（白背景）
  • Top: channel banner image (background)
  • Bottom: avatar + title + handle (white background)

クリックすると別ウィンドウ（target="_blank"）でチャンネルが開きます。

Clicking opens the channel in a new window (target="_blank").

画像URLの取得方法 How to get image URLs

YouTubeはページをJavaScriptで動的生成するため、自動取得は困難です。ブラウザで以下の手順で取得してください。

YouTube pages are dynamically generated, so automatic URL extraction is not reliable. Use your browser:

  1. チャンネルページ（https://www.youtube.com/@チャンネル名）を開く
  2. バナー画像 or アイコン画像を右クリック →「画像アドレスをコピー」
  3. yt-banner.js の対応するURLを置き換える
  1. Open the channel page (https://www.youtube.com/@channelname)
  2. Right-click the banner or icon image → "Copy image address"
  3. Replace the corresponding URL in yt-banner.js

レッスンページの追加 Adding a Lesson Page

新しい日別ページを追加する手順です。例として Month 4 Lesson 1（m4-1.html）を追加します。

Steps to add a new day page. Example: adding Month 4 Lesson 1 (m4-1.html).

① 既存ページをコピーして作成する ① Copy an existing page

最も近い構成のページ（例：m3-6.html）をコピーして m4-1.html にリネームします。

Copy the closest existing page (e.g. m3-6.html) and rename it to m4-1.html.

② ヘッダーとサブヘッダーを書き換える ② Update header and subheader 

<!-- title -->
<title>Month 4 Day 1〜4 | Extra English</title>

<!-- ヘッダー -->
<a href="month4.html" class="back-btn">‹</a>
<div class="header-title">Month 4 · レッスン 1</div>

<!-- Dayバッジ -->
<span class="day-badge">Day 1〜4</span>

<!-- YouTubeバナー -->
<div class="yt-banner" data-day="Month 4 Day 1〜4"></div>

③ コンテンツを書き換える ③ Replace content

PDFの該当箇所を参照して、.listen-card 内のリスニングポイントと .accordion-list 内の学習ポイントを書き換えます。

Referring to the PDF, replace the listening points in .listen-card and the learning points in .accordion-list.

④ ナビゲーションを設定する ④ Set navigation 

<div class="page-nav">
  <a href="m3-6.html" class="nav-btn nav-prev">‹ 前へ</a>  <!-- 前のページ -->
  <span class="nav-btn nav-next disabled">次へ ›</span>    <!-- 次がない場合 -->
</div>

⑤ 月別ページ（month4.html）にカードを追加する ⑤ Add a card to the month page 

<div class="lesson-grid">
  <a href="m4-1.html" class="lesson-card">
    <div class="lesson-num">1</div>
    <div class="lesson-card-body">
      <div class="lesson-card-day">Day 1〜4</div>
      <div class="lesson-card-title">I told you / Come on ...</div>
      <div class="lesson-card-count">学習X点 ｜ リスニングX点</div>
    </div>
    <div class="lesson-card-arrow">›</div>
  </a>
</div>

⑥ list.html に内容を追加する ⑥ Add content to list.html

list.html の Month 4 グループ（data-month="m4"）に同じリスニング・学習ポイントを追加します。フィルターボタンも必要なら追加してください。

Add the same content to a Month 4 group (data-month="m4") in list.html. Add a filter button if needed.

日別ページの2カラム構造 Two-column layout on day pages

SP は縦積み、タブレット以上は左：学習ポイント／右：リスニングポイントの2カラムになります。

SP stacks vertically; tablet+ shows a 2-column layout: learning (left) / listening (right).

<div class="day-cols">
  <div class="day-col-left">   <!-- リスニングポイント（DOM上は先頭）-->
    ...
  </div>
  <div class="day-col-right">  <!-- 学習ポイント（CSS order:-1 で常に左表示）-->
    ...
  </div>
</div>
表示順について：学習ポイントは CSS の order: -1 によって常に左（SP では上）に表示されます。DOM 上の順序（day-col-left が先）は変えないでください。
Display order: Learning points always appear left (or top on SP) via CSS order: -1. Do not change the DOM order (day-col-left first).

運用上の注意 Important Notes

検索エンジンへの非表示設定 Search engine exclusion

全ページに以下が設定されており、検索エンジンにインデックスされません。意図的な設定です。変更しないでください。

All pages include the following tag to prevent search engine indexing. This is intentional — do not remove it.

<meta name="robots" content="noindex,nofollow">

右クリック禁止 Right-click disabled

右クリック（コンテキストメニュー）は yt-banner.js 内のイベントリスナーで全ページ無効化されています。意図的な設定です。

Right-click (context menu) is disabled on all pages via an event listener in yt-banner.js. This is intentional.

document.addEventListener("contextmenu", function(e) { e.preventDefault(); });

コンテンツの変更・削除について Modifying or removing content

原稿PDFが正本です。
HTMLのコンテンツを変更・削除する場合は必ずPDFと照合してください。誤りを発見した場合はユーザー（管理者）に確認してから修正してください。
The PDF is the canonical source.
Always cross-reference the PDF when editing HTML content. If you find an error, confirm with the administrator before correcting it.

list.html との同期 Syncing with list.html

日別ページと list.html は自動同期されません。
どちらかを変更した場合は、もう一方も同じ内容に手動で更新してください。
Day pages and list.html are not automatically synced.
Any change to one must be manually applied to the other.
