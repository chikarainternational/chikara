レッスンデータの構造 Lesson Data Structure

各ファイルの <script> 冒頭にある lessons 配列がすべてのコンテンツ定義です。

The lessons array at the top of the <script> block in each file defines all content.

const lessons = [
  {
    no: 1,
    items: [
      { id: 'A1', v: 'eiOJrUijqb4', t: '母音「A」の発音' },
      { id: 'I2', v: 'kNCYzCI7Wp8', t: '母音「I」の発音' }
    ]
  },
  ...
];
プロパティProperty 役割Role 例Example
no レッスン番号（1始まりの連番）Lesson number (sequential, starting at 1) 1
id 動画カードのHTML id。目次リンクのアンカーにもなる。全動画でユニークにすることHTML id for the video card. Also used as the TOC anchor. Must be unique across all videos. 'A1'
v YouTubeの動画ID（URLの ?v= 以降）YouTube video ID (the part after ?v= in the URL) 'eiOJrUijqb4'
t 動画タイトル（目次・カードに表示）Video title (shown in TOC and card) '母音「A」の発音'

YouTube動画IDの確認方法 How to find a YouTube Video ID

YouTubeのURLが https://www.youtube.com/watch?v=eiOJrUijqb4 の場合、v= 以降の文字列 eiOJrUijqb4 が動画IDです。

If the YouTube URL is https://www.youtube.com/watch?v=eiOJrUijqb4, the video ID is the string after v=: eiOJrUijqb4.

動画の差し替え Replacing Videos

対象の items 内で v の値を新しい動画IDに書き換えるだけです。

Simply replace the v value with the new video ID in the target items entry.

// 変更前
{ id: 'A1', v: 'eiOJrUijqb4', t: '母音「A」の発音' }

// 変更後（動画IDを差し替え）
{ id: 'A1', v: '新しいVideoId', t: '母音「A」の発音' }
// Before
{ id: 'A1', v: 'eiOJrUijqb4', t: '母音「A」の発音' }

// After (replace video ID)
{ id: 'A1', v: 'newVideoId', t: '母音「A」の発音' }
タイトル（t）も変更する場合は忘れずに。サムネイルも自動的に新しい動画のものに切り替わります。
Remember to update the title (t) if it also changes. The thumbnail will automatically update to match the new video.

レッスンの追加・削除 Adding / Removing Lessons

レッスンを追加する Adding a lesson

lessons 配列の末尾に新しいオブジェクトを追加します。

Append a new object to the end of the lessons array.

{ no: 25, items: [{ id: 'XY47', v: 'newVideoId', t: 'New Lesson Title' }] },
id のルール：英数字で構成し、全動画でユニークにしてください。慣例として「音素の略称＋連番」（例: A1, TH17, ABC44）としています。
id rules: Use alphanumeric characters, unique across all videos. Convention is "phoneme abbreviation + sequence number" (e.g. A1, TH17, ABC44).

1つのレッスンに複数動画を入れる Multiple videos per lesson 

{ no: 25, items: [
  { id: 'XX47', v: 'VideoId_1', t: 'Title 1' },
  { id: 'YY48', v: 'VideoId_2', t: 'Title 2' }
]}

レッスンを削除する Removing a lesson

該当行を配列から削除します。no（レッスン番号）は連番を維持してください。途中の番号が抜けると目次の番号表示がずれます。

Delete the row from the array. Keep no sequential — gaps in numbering will misalign the TOC display.

ヘッダーの動画本数表示を更新する Updating the header count

レッスン数・動画数を変更したら、各ファイルのヘッダー表示も手動で合わせます。

After changing lesson/video counts, manually update the header text in each file.

<p>LESSON 1–24 / 46 VIDEOS</p>

特定レッスンへの直リンク Direct Links to Lessons

URLにハッシュを付けると、ページ読み込み時に該当アコーディオンが自動で開きます。

Adding a hash to the URL auto-opens the corresponding accordion on page load.

形式Format 説明Description 例Example
#L{番号number} レッスン番号で指定Open by lesson number page/1.html#L5 → LESSON 5を開くOpens LESSON 5
#{動画idvideo id} 動画IDで指定Open by video ID page/1.html#DEF45 → その動画を含むレッスンを開くOpens the lesson containing that video

運用上の注意 Important Notes

検索エンジンへの非表示設定 Search engine exclusion

全ページに以下が設定されており、検索エンジンにインデックスされません。意図的な設定です。変更しないでください。

All pages include the following tag to prevent search engine indexing. This is intentional — do not remove it.

<meta name="robots" content="noindex,nofollow">

右クリック・開発者ツールの無効化 Disabled interactions

以下の操作はJSで無効化されています。これも意図的な設定です。

The following are intentionally disabled via JavaScript.

  • 右クリック（コンテキストメニュー）
  • F12（開発者ツール）
  • Ctrl+C / Ctrl+A / Ctrl+U / Ctrl+S / Ctrl+P
  • Right-click (context menu)
  • F12 (DevTools)
  • Ctrl+C / Ctrl+A / Ctrl+U / Ctrl+S / Ctrl+P

YouTubeの埋め込みドメイン YouTube embed domain

youtube-nocookie.com を使用しています（プライバシー強化モード）。youtube.com には変更しないでください。

Uses youtube-nocookie.com (privacy-enhanced mode). Do not change to youtube.com.

変更は3ファイルに同時適用してください。
1.html / 2.html / 3.html は同じデータ・ロジックを持ちます。1ファイルだけ変更するとテーマ間で内容が食い違います。
Apply changes to all 3 files simultaneously.
1.html / 2.html / 3.html share the same data and logic. Updating only one file will cause content inconsistency between themes.
